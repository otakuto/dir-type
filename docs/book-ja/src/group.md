# グループによる複数エントリのレコード化

`group:` マーカーキーを持ち、`dir:` / `file:` / `rule:` を持たないエントリを**グループ**と呼ぶ。`id:` と `::` を兄弟キーとして持ち、複数のエントリをひとまとまりのレコードとして同じ階層に束ねる。

## ディレクトリ構成

```text
<!-- cmdrun cd ../../../ && tree tutorials/group --noreport -->
```

## ルール定義

```yaml
{{#include ../../../tutorials/group/.dir-type.yaml}}
```

## 解説

通常の `dir:` エントリはサブディレクトリを1つ挟んでエントリをまとめる。グループはディレクトリを挟まずに、同じ階層のエントリを1つのレコード `unit` として束ねる。

この例では `name.txt`（ファイル）と `data/`（ディレクトリ、その中に `value.txt` を持つ）が、`id: unit` という名前のグループとして1つのレコードに束ねられている。

グループの主な用途は `for` でのイテレーションである。`for` でグループを展開すると、グループ内の各エントリの収集結果に対して繰り返し処理を適用できる。応用パターンは `016_group-clone` で確認できる。
