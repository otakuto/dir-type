# `dir` と `regex` によるパターンマッチ

`dir:` も `file:` と同様に、値をマッピングにして `regex:` キーに正規表現を書くことで、名前のパターンでディレクトリを受理できる。正規表現にマッチするディレクトリをすべて受理し、それぞれに `::` で指定した内容を適用する。

## ディレクトリ構成

```text
<!-- cmdrun cd ../../../ && tree -a tutorials/dir-regex --noreport -->
```

## ルール定義

```yaml
{{#include ../../../tutorials/dir-regex/.dir-type.yaml}}
```

このルールは `a/` / `b/` / `c/` のような 1 文字の小文字英字ディレクトリを全て受理し、それぞれの中に `file` という名前のファイルが存在することを要求する。

`dir:` + `regex:` を使うと、**パターンにマッチするすべてのディレクトリに同じルールを適用**できる。ディレクトリ名が動的に決まる構成(バージョン番号やモジュール名のディレクトリが並ぶ場合など)でも、一度の宣言で全体を型付けできる。

名前付きキャプチャグループ(`(?<name>...)`)と `id:` を組み合わせると、マッチしたディレクトリ群をコレクションとして収集し、後続の `for` でイテレートできる。詳細は[`for` によるコレクションの反復](./for.md)を参照のこと。
