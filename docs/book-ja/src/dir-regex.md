# `dir` + `regex` — パターンを持つディレクトリ型

`dir:` も `file:` と同様に、値をマッピングにして `regex:` キーに正規表現を書くことで、名前のパターンでディレクトリを受理できる。正規表現にマッチするディレクトリを全て受理し、それぞれに `::` で指定した内容を適用する。

## ディレクトリ構成

```text
<!-- cmdrun cd ../../../ && tree tutorials/dir-regex --noreport -->
```

## ルール定義

```yaml
{{#include ../../../tutorials/dir-regex/.type-dir.yaml}}
```

このルールは `a/` / `b/` / `c/` のような 1 文字の小文字英字ディレクトリを全て受理し、それぞれの中に `file` という名前のファイルが存在することを要求する。

`dir:` + `regex:` の強力な点は、**パターンにマッチする全ディレクトリに同じルールを適用する**ところにある。ディレクトリ名が動的に決まるような構成——たとえばバージョン番号やモジュール名のディレクトリが並ぶ場合——でも、一度の宣言で全体を型付けできる。

名前付きキャプチャグループ(`(?<name>...)`)と `id:` を組み合わせると、マッチしたディレクトリ群をコレクションとして収集し、後続の `for` でイテレートできる。詳細は[for — コレクションをイテレートする](./for.md)を参照のこと。
