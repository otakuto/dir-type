# `one_of` によるちょうど 1 つの要求

**`one_of`** は配下に列挙したエントリのうち、ちょうど 1 つだけが存在することを要求する。両方存在した場合も、どちらも存在しない場合もエラーになる。

## ディレクトリ構成

```text
<!-- cmdrun cd ../../../ && tree tutorials/one_of --noreport -->
```

## ルール定義

```yaml
{{#include ../../../tutorials/one_of/.dir-type.yaml}}
```

このルールは `a/` や `b/` のような各ディレクトリが `left.txt` か `right.txt` のどちらか一方だけを持つことを要求する。両方あってもエラー、どちらもなくてもエラーになる。

`one_of:` の配下には任意のエントリ型を並べられる。`file:` だけでなく `dir:` や `optional:` を組み合わせることもできる。`id:` を付けたエントリを `one_of:` の中で使うと、どちらの選択肢が選ばれたかを後続の `match` で分岐できる。
