# `one_of` — ちょうど1つを要求する

「どちらかの形式で書かれていること、しかし両方は禁止」という規約を宣言したい場面がある。そのときは `one_of` を使う。配下に列挙したエントリのうち、ちょうど 1 つだけが存在していなければならない。

両方存在したり、どちらも存在しない場合はエラーになる。

## ディレクトリ構成

```text
<!-- cmdrun cd ../../../ && tree tutorials/one_of --noreport -->
```

## ルール定義

```yaml
{{#include ../../../tutorials/one_of/.dir-lint.yaml}}
```

このルールは `a/` や `b/` のような各ディレクトリが `left.txt` か `right.txt` のどちらか一方だけを持つことを要求する。両方あってもエラー、どちらもなくてもエラーになる。

`one_of:` の配下には任意のエントリ型を並べられる。`file:` だけでなく `dir:` や `optional:` を組み合わせることもできる。また `id:` を付けたエントリを `one_of:` の中で使うと、どちらの選択肢が選ばれたかを後続の `match` で分岐できる。
