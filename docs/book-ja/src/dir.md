# `dir` — サブディレクトリを必須とする

`dir` エントリは、指定した名前のディレクトリが存在することを要求する。`::` でネストすることで、そのサブディレクトリ内のルールも続けて記述できる。ディレクトリが存在しなければ `LT002 required name not found` を報告する。

## ディレクトリ構成

```text
<!-- cmdrun cd ../../../ && tree tutorials/dir --noreport -->
```

## ルール定義

```yaml
{{#include ../../../tutorials/dir/.type-dir.yaml}}
```

このルールは、`src/` ディレクトリが存在し、その中に `main.txt` が存在することを要求する。`dir:` の直後に `::` を置くことで、そのディレクトリ配下の構造をインラインで記述できる。

`dir:` エントリは階層的にネストできる。深い階層の構造も、インデントを深くするだけで表現できる。`::` で列挙されていないエントリがディレクトリ内に存在した場合は `LT001 undeclared path` として報告される。
