# `file` と `regex` によるパターンマッチ

ファイル名が動的に変わる場合は正規表現で受理できる。`file:` の値をスカラーではなくマッピングにし、`regex:` キーに正規表現を書けばよい。

正規表現にマッチするファイルを 0 個以上受理する。存在するファイルをすべて `LT001 undeclared path` にしないためのパターンとして有用だ。

## ルール定義

```yaml
{{#include ../../../tutorials/file-regex/.type-dir.yaml}}
```

このルールは `0.txt` / `1.txt` / `2.txt` のような `<数字>.txt` 形式のファイルを全て受理する。正規表現は Rust の `regex` クレートの構文に従う。

名前付きキャプチャグループ(`(?<name>...)`)を使うと、マッチした部分文字列を後続の `for` や `match` で参照できる。たとえば `'^(?<stem>[a-z]+)\.txt$'` と書けば、`stem` というキャプチャ名でファイル名のベースを取り出せる。
